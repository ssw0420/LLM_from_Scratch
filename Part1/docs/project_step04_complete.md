Step 4 트라이그램 모델이 완료되었다.

**바이그램 → 트라이그램 핵심 변경:**
- 키: 단일 코드포인트 → (cp1, cp2) 쌍
- 해시 함수: hash_cp(cp) → hash_pair(cp1, cp2) (두 코드포인트 결합)
- 상태 추적: prev_cp 1개 → prev2, prev1 2개 (슬라이딩 윈도우)
- 해시 테이블 크기: 8,192 → 131,071 (행이 ~22배 증가)
- BOS 패딩: 바이그램은 BOS 1개, 트라이그램은 BOS 2개 (N-gram은 N-1개)

**파일 구성:**
1. step04_trigram.c — BOS/EOS 통합 + CSV 출력 + 사전 검증 (30개)
2. step04_trigram_1000.c — 동일 구조, 1000개 대량 생성 검증 (콘솔에는 요약만 출력)

**검증 결과:**
- 30개: 77~93% 적중 (표본 작아서 분산 큼)
- 1000개: 77.3% 적중 (안정적 수치)
- 바이그램 대비: ~50% → ~77% 으로 향상

**통계:**
- 고유 행(cp1,cp2 쌍): 129,730
- 총 트라이그램 쌍: 2,273,216
- CSV 행 수: 360,575행

**컴파일 시 주의:**
- `/utf-8` 플래그 필수 — 없으면 fprintf의 한글 문자열 리터럴이 CP949로 인코딩되어 출력 파일에서 깨짐

바이그램 → 트라이그램으로 컨텍스트 확장, 성능 향상을 확인한 단계.
