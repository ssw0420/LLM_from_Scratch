## Step 5: 글자 단위 N-gram 문장 생성

Step 4까지는 사전 표제어(words_only.txt)로 **단어**를 생성했다. Step 5에서는 소설 코퍼스(data_all.txt)로 **문장**을 생성하는 것으로 확장.

### 핵심 변경점 (Step 4 → Step 5)
- 학습 데이터: `words_only.txt` (사전) → `data/data_all.txt` (소설 35,034문장, 190만 글자)
- 토큰 범위: 한글 완성형만 → 모든 코드포인트 (공백, 쉼표, 마침표 등 동등 처리)
- BOS/EOS: 단어 경계 → 문장 경계 (각 줄 = 1문장)
- N-gram 차수: 고정(tri) → 가변(3, 5, 7) 비교
- 자료구조: `TrigramRow.cp1,cp2` → `NgramRow.context[MAX_ORDER-1]` 일반화
- 해시: `hash_pair(cp1,cp2)` → `hash_context(ctx[], len)` FNV-1a

### 문장 생성 과정 (5단계 파이프라인)

1. **데이터 로드**: 각 줄을 UTF-8 디코딩하여 코드포인트 배열로 변환. 공백(32), 쉼표(44), 마침표(46) 등도 동등한 토큰.
2. **BOS/EOS 패딩**: 각 문장 앞에 (N-1)개 BOS, 뒤에 EOS 1개. 예) 3-gram + "가 나" → `[BOS, BOS, 가, 공백, 나, EOS]`
3. **슬라이딩 윈도우 카운팅**: 길이 N 윈도우를 1칸씩 밀며, 앞 (N-1)글자=컨텍스트, 마지막 1글자=다음글자로 분리하여 해시맵에 빈도 축적. 예) `[BOS, BOS]→가`, `[BOS, 가]→공백`, `[가, 공백]→나`, `[공백, 나]→EOS`
4. **누적 확률 구축**: 각 컨텍스트의 빈도를 누적 확률로 변환 → 이진 탐색으로 O(log n) 샘플링 가능
5. **문장 생성**: 컨텍스트를 BOS (N-1)개로 초기화 → 해시맵 조회 → 누적 확률에서 랜덤 샘플링 → 컨텍스트 왼쪽 시프트 → EOS 나올 때까지 반복

**고유 컨텍스트 수** = 해시맵의 행(row) 수 = 학습에서 발견된 서로 다른 (N-1)글자 조합 수. N이 커질수록 폭발적으로 증가하며 각 빈도는 1~2회로 떨어짐(희소) → 원문 복사(과적합).

### 결과
| N | 컨텍스트 수 | 품질 |
|---|---|---|
| 3 | 37,921 | 단어 경계 불안정, 문법 파괴 |
| 5 | 480,542 | 어절 자연스러움, 절 간 연결 끊김 |
| 7 | 1,146,338 | 원문급이지만 과적합(암기) |

### 논의된 사항
- 괄호/따옴표 유형별로 데이터 분리 학습은 N-gram에서 비효율적 (데이터 쪼개짐 → 희소성 증가)
- 데이터양 증가도 N-gram 근본 한계 해결 불가 (한글 조합 경우의 수 >> 관측 가능 컨텍스트)
- 조건부 생성, 일반화는 신경망 단계에서 의미 있음
